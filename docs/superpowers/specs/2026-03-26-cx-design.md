# cx Design Spec

macOS CLI for managing Apple Contacts via JXA. Replaces v1 Swift/CNContactStore implementation, which cannot access contact notes due to Apple entitlement restrictions.

## Decisions

| Decision          | Choice                                             | Rationale                                                               |
| ----------------- | -------------------------------------------------- | ----------------------------------------------------------------------- |
| Language          | JXA via `osascript -l JavaScript`                  | Full Contacts.app access, native JSON handling, no build step           |
| File structure    | Single `cx.js` file                                | JXA has no module system; at ~500 lines, one file is manageable         |
| Output format     | Text only                                          | Claude Code parses text fine; JSON adds complexity for no real consumer |
| Search            | Contacts.app built-in search                       | Notes search is not critical; built-in is faster and simpler            |
| Multi-value input | Repeated flags with `label:value` + `--json` stdin | Simple cases stay simple; complex cases pipe JSON                       |
| Linting           | shellcheck + shfmt for shell scripts only          | JXA dialect causes linter noise that exceeds value                      |
| Task runner       | Taskfile.yml                                       | Consistent with user's other repos                                      |

## File Structure

```text
cx/
├── cx                # Bash wrapper (entry point)
├── cx.js             # All JXA logic
├── Taskfile.yml      # install, test, lint, fmt
├── tests/
│   └── test.sh       # Integration tests
├── cx-spec.md
├── CLAUDE.md
└── README.md
```

## Entry Point

`cx` is a bash script that resolves its real path (following symlinks), then runs:

```bash
osascript -l JavaScript "$(dirname "$0")/cx.js" -- "$@"
```

CLI args are available in JXA via `ObjC.unwrap($.NSProcessInfo.processInfo.arguments)`.

## Commands

### cx list

```text
cx list [--group <name>]
```

Lists all contacts, or contacts in a group. Output: columnar table (name, email, phone, organization).

### cx search

```text
cx search <query>
```

Uses Contacts.app's built-in search. Does not search notes content. Output: same table format as list.

### cx get

```text
cx get <id>
```

Full contact details by short ID (first 8 chars) or full UUID. Output: card layout with labeled fields, including notes.

### cx create

```text
cx create --first <name> --last <name> [--email [label:]<addr>] [--phone [label:]<num>] [--org <name>] [--title <title>] [--note <text>] [--group <name>]
cx create --json  (reads JSON from stdin)
```

Repeated flags for multi-value fields. Label defaults to "home" if omitted. `--json` reads a full contact object from stdin; mutually exclusive with flags.

Output: one-line confirmation with ID.

### cx update

```text
cx update <id> [--first <name>] [--last <name>] [--note <text>] [--email [label:]<addr>] ...
cx update <id> --json  (reads JSON from stdin)
```

Only specified fields are changed. Same flag and `--json` semantics as create.

Output: one-line confirmation.

### cx delete

```text
cx delete <id> [--force]
```

Without `--force`: prints contact summary, exits with code 5 (confirmation required). With `--force`: deletes and prints one-line confirmation.

### cx groups

```text
cx groups list
cx groups members <name>
cx groups add <contact-id> <group-name>
cx groups remove <contact-id> <group-name>
cx groups create <name>
cx groups delete <name> [--force]
```

## ID Resolution

Short IDs are the first 8 characters of the contact UUID. If a short ID matches multiple contacts, exit with code 4 and list the ambiguous matches.

## Error Handling

- All errors go to stderr as text
- Stdout is reserved for data
- No silent failures

### Exit Codes

| Code | Meaning                                         |
| ---- | ----------------------------------------------- |
| 0    | Success                                         |
| 1    | General error                                   |
| 2    | Permission denied (Contacts access not granted) |
| 3    | Not found                                       |
| 4    | Ambiguous ID                                    |
| 5    | Confirmation required (delete without --force)  |

## Contact Properties

All readable and writable:

| Property          | CLI flag          | Notes                     |
| ----------------- | ----------------- | ------------------------- |
| First name        | `--first`         |                           |
| Last name         | `--last`          |                           |
| Middle name       | `--middle`        |                           |
| Name prefix       | `--prefix`        |                           |
| Name suffix       | `--suffix`        |                           |
| Nickname          | `--nickname`      |                           |
| Maiden name       | `--maiden`        |                           |
| Organization      | `--org`           |                           |
| Job title         | `--title`         |                           |
| Department        | `--dept`          |                           |
| Note              | `--note`          | Core capability           |
| Birthday          | `--birthday`      | ISO 8601 or --MM-dd       |
| Emails            | `--email`         | Repeatable, `label:value` |
| Phones            | `--phone`         | Repeatable, `label:value` |
| Addresses         | via `--json` only | Too complex for flags     |
| URLs              | `--url`           | Repeatable, `label:value` |
| Social profiles   | via `--json` only |                           |
| Related names     | `--related`       | Repeatable, `label:value` |
| Instant messaging | via `--json` only |                           |
| Dates             | `--date`          | Repeatable, `label:value` |

## Testing

Integration test script (`tests/test.sh`) covering the full lifecycle:

1. Create a test contact with known values including a note
2. Search for it by name — verify it appears
3. Get by ID — verify all fields including note
4. Update the note — get again, verify change
5. Delete with `--force` — verify gone
6. Group lifecycle: create group, add contact, list members, remove, delete group

Test contacts use a `CxTest_` prefix. A trap on EXIT ensures cleanup even on failure.

## Taskfile

| Task             | Command                                    | Purpose           |
| ---------------- | ------------------------------------------ | ----------------- |
| `task install`   | Symlink `cx` to `~/.local/bin/cx`          | Installation      |
| `task uninstall` | Remove symlink                             | Removal           |
| `task test`      | Run `tests/test.sh`                        | Integration tests |
| `task lint`      | `shellcheck` + `shfmt -d` on shell scripts | Static analysis   |
| `task fmt`       | `shfmt -w` on shell scripts                | Auto-format       |

## Scope Boundaries

**In scope:** Full CRUD on contacts and groups, all properties including notes, text output, short ID resolution, search, `--json` stdin for complex input.

**Out of scope:** Contact images, vCard import/export, sync/conflict resolution, batch operations, searching within notes content, JSON output format.
